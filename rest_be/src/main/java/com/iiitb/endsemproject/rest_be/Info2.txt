What is Optional<T>?
Optional<T> is a container object that may or may not contain a non-null value of type T.

Think of it as a small, labeled box that can hold one item:

Scenario A: The item is inside the box. (The value exists.)

Scenario B: The box is empty. (The value is null/not found.)

The goal of Optional is to eliminate the need for those dangerous, error-prone checks like if (result == null) everywhere in your code.

Initial State,Final Result (Return Type: Optional<Path>)
Optional<SalaryDisbursement> contains a record,Optional<Path> contains the valid file path
Optional<SalaryDisbursement> is empty,"Optional<Path> is empty (No NPE, returns safely)"

This single line of code safely transforms an object retrieved from the database into a secure file path object, avoiding crashes if the data is missing.

Here's a simple breakdown of the process:

disbursement.map(...): This checks the Optional<SalaryDisbursement> container. If the salary record is present, it proceeds; otherwise, it stops immediately and returns an empty result.

SalaryDisbursement::getPayslipFilePath: If the record is present, this step extracts the String containing the file path (e.g., "C:/payslips/oct.pdf").

.map(Path::of): If the file path string is present, this final step converts that simple string into a secure Java Path object, which is needed to access the actual file on the server.

The final result is an Optional<Path> that either contains the secure file path or is empty, preventing a NullPointerException if the record was not found in the database.
